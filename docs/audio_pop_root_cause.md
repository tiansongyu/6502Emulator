# NES 模拟器声音爆音问题 —— 根因分析报告

- 仓库：`6502Emulator`
- 涉及模块：`Nes2A03`（APU）、`Bus::clock`（采样降频）、`olcPGEX_Sound`（ALSA 输出线程）
- 现象：游戏运行时音频持续出现"咔嗒/爆音"，且随运行时间增长而恶化

---

## 1. 音频信号链总览

```
NES 主时钟 21.477 MHz
         └─ /6 ──► APU 时钟 ~894.886 kHz ──► Nes2A03::clock()
                                                  │
                                                  ▼
                                       生成 pulse1/pulse2/noise 样本
                                                  │
                                                  ▼
Bus::clock() 累加 dAudioTime ──► 命中 1/44100 s ──► 取一次 GetOutputSample()
                                                  │
                                                  ▼
                            olcPGEX_Sound::AudioThread (ALSA, 44.1kHz mono)
```

关键点：

- 上游每秒产生约 **89 万个**样本，下游消费 **44 100 个**样本，降采样比约 **20:1**。
- 降采样在 `Bus::clock` 里只是"到点取一个"，**没有任何抗混叠滤波**。
- 任何在 APU 端产生的波形阶跃 / 相位跳变，都会被点采样直接搬到扬声器上变成爆音。

---

## 2. 五个互相叠加的根因

### 2.1 根因 A：傅里叶级数振荡器存在相位跳变（**直接导致爆音**）

源码（修复前 `Nes2A03.h::oscpulse::sample`）：

```cpp
for (double n = 1; n < harmonics; n++) {
  double c = n * frequency * 2.0 * pi * t;   // ★
  a += -approxsin(c) / n;
  b += -approxsin(c - p * n) / n;
}
```

问题：
- `c = n · frequency · 2π · t` 是**用当前 frequency 直接乘绝对时间 t**，而不是对相位做积分。
- 一旦 CPU 写入寄存器修改 `pulse_seq.reload`（每次换音都会发生），`frequency = 1789773 / (16·(reload+1))` 立刻改变。
- 由于相位用的是 `n · f_new · t`，而上一刻还在用 `n · f_old · t`，**所有 20 个谐波的瞬时相位同时整体跳变**。
- 每次跳变 ⇒ 一次明显的波形不连续 ⇒ 一次"咔哒声"。

> 这是听到的爆音的**主要来源**。在 SMB 之类换音密集的游戏里几乎每一帧都在产生爆音。

---

### 2.2 根因 B：浮点精度随运行时间崩塌（**爆音随时间恶化**）

```cpp
dGlobalTime += (0.3333333333 / 1789773);   // 每次 APU clock 都在累加，永不归零

auto approxsin = [](float t) {              // ★ 形参是 float
  float j = t * 0.15915;
  j = j - static_cast<int>(j);              // ★ 大数转 int
  return 20.785f * j * (j - 0.5) * (j - 1.0f);
};
```

问题：
- 调用方 `c` 是 `double`，传进 `approxsin(float t)` 时被截断为单精度。
- `dGlobalTime` 单调递增，运行几十秒后 `c = n·f·2π·t` 在 `float` 下精度严重不足。
- `j - static_cast<int>(j)` 取小数部分时，当 `j` 超出 `int` 表示范围时是**未定义行为**；即使没溢出，单精度的相位误差也会让正弦近似输出近似随机数。
- 表现：刚开机声音相对正常，玩一会儿之后越来越脏、越来越多杂音。

---

### 2.3 根因 C：静音瞬间的硬归零 + 直流偏置（**每次通道开关一次咔哒声**）

```cpp
if (pulse1_lc.counter > 0 && pulse1_seq.timer >= 8 &&
    !pulse1_sweep.mute && pulse1_env.output > 2)
  pulse1_output += (pulse1_sample - pulse1_output) * 0.5;
else
  pulse1_output = 0;          // ★ 直接拉到 0，没有平滑
```

```cpp
return ((1.0 * pulse1_output) - 0.8) * 0.1 +
       ((1.0 * pulse2_output) - 0.8) * 0.1 +
       ((2.0 * (noise_output - 0.5))) * 0.1;
```

问题：
- 通道激活时，`pulse_output` 在 0 附近震荡；通道关闭瞬间，被强制拉到 0。
- `GetOutputSample` 里又做了 `(x - 0.8) * 0.1` 的偏置：
  - 通道**激活**时：输出 = `(波形 - 0.8) * 0.1`，中心约 `-0.08`。
  - 通道**关闭**时：输出 = `(0 - 0.8) * 0.1 = -0.08`（还有另两个通道的偏置叠加）。
- 也就是说：每次有通道开/关，混音输出会出现**直流台阶**。
- 直流台阶在扬声器里 = 砰的一声。

---

### 2.4 根因 D：没有抗混叠滤波

- APU 端原始采样率约 894.886 kHz，下游需要 44.1 kHz。
- 修复前的代码在 APU 端只生成了高谐波丰富的波形，然后**直接点采样**给 ALSA。
- 任何高于 22.05 kHz 的能量都会折叠回可听频段，叠加成可感知的"沙沙声"和"颗粒感"。
- 这部分不是单次"啪"的爆音，而是把整体音质拖脏的底噪。

---

### 2.5 根因 E：硬件正确路径被弃用（设计层面问题）

```cpp
bool bUseRawMode = false;   // 永远 false
...
//  if (bUseRawMode)
{
  pulse1_seq.clock(...);    // 真实硬件序列器输出
}
// else
{
  // 全部走有问题的傅里叶路径
  pulse1_sample = pulse1_osc.sample(dGlobalTime);
}
```

- 真实 NES 硬件的方波就是 **8 位旋转模板的 LSB**，本就是离散方波，**不需要**傅里叶重建。
- 代码里同时维护了两条路径，但实际进入扬声器的永远是有问题的傅里叶路径。
- 这是上面所有问题的设计层面诱因 —— 选错了模型。

## 3. 五个根因之间的关系

| 根因 | 触发条件 | 听感 |
| ---- | -------- | ---- |
| A 相位跳变 | 每次换音 / 写 0x4002 / 0x4003 等 | 频繁咔哒、爆音（**最显眼**） |
| B 浮点漂移 | 持续运行 | 越玩越脏、越糊 |
| C 硬归零 + 直流偏置 | 每次通道 enable/disable、长度计数器到 0 | 每次开关一次"砰" |
| D 无抗混叠 | 一直存在 | 底噪、颗粒感 |
| E 模型选错 | 一直存在 | 是 A/B 的根 |

A、B、C 都在反复地往输出里注入**阶跃 / 不连续**，再叠加 D 没有低通滤波 —— 所有阶跃以最大力度被搬到扬声器，于是出现持续爆音。

---

## 4. 修复方案（已实施）

设计原则：**用真实硬件信号替换错误模型，并在合适的位置做正确的滤波。**

1. **删除傅里叶振荡器和 `dGlobalTime`**
   - 移除 `oscpulse::sample(t)` 整段实现，移除 `bUseRawMode` 死代码。
   - 杜绝根因 A、B、E。

2. **改用硬件序列器输出**
   - 每个 APU 时钟，先 `seq.clock(...)` 推进序列器；
   - 通道样本 = `seq.output × env.output`，再用 `enable / length / sweep / timer` 进行门控；
   - 这是真实 2A03 的输出方式。

3. **APU 端一阶低通滤波（一通道一个）**
   ```cpp
   y += alpha * (x - y);   // alpha ≈ 0.0961, fc ≈ 14 kHz @ 894.886 kHz
   ```
   - 解决根因 D：在降采样到 44.1 kHz **之前**完成抗混叠；
   - 顺带把序列器跳变（C 的一部分）平滑掉，不再产生硬阶跃。

4. **混音用标准 NES 线性近似**
   ```
   pulse_out = 0.00752 · (p1 + p2)
   tnd_out   = 0.00494 · noise        // triangle/dmc 暂未实现
   mixed     = pulse_out + tnd_out
   ```
   - 来自 nesdev wiki，是真实非线性混音器的线性近似；
   - 各通道贡献量级一致，不会再出现"某通道独占输出"的情况。

5. **混音后一阶高通滤波（DC blocker）**
   ```cpp
   y[n] = x[n] - x[n-1] + R · y[n-1];   // R = 0.9985
   ```
   - 解决根因 C：任何通道开/关引起的直流台阶都会被高通快速消除，不会再变成扬声器里的砰声。

## 6. 教训与可复用经验

1. **音频里的"爆音"几乎总是阶跃问题**。听到爆音先去找：
   - 相位跳变（频率改变时是否对相位做积分？）；
   - 直流台阶（开关通道有没有平滑？混音是否引入了与状态相关的偏置？）。
2. **任何降采样之前必须有抗混叠滤波**。这是 DSP 的硬性约束，不存在"先点采样看看效果"的中间状态。
3. **不要混用"建模信号"和"硬件信号"两条路径**。NES 方波本身就是离散信号，硬件路径是更短、更准、更便宜的实现。
4. **绝对时间累积是浮点炸弹**。连续运行的模拟器里，凡是 `t += dt` 这种累加全局时间被乘进相位的写法，都要换成相位累加 `phase += dphi` 并按需取模 `2π`。
